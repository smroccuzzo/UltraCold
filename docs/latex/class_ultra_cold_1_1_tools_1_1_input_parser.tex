\doxysubsection{Detailed Description}
Class to read input parameters from files. 

\begin{DoxyAuthor}{Author}
Santo Maria Roccuzzo (\href{mailto:santom.roccuzzo@gmail.com}{\texttt{ santom.\+roccuzzo@gmail.\+com}})
\end{DoxyAuthor}
The \mbox{\hyperlink{class_ultra_cold_1_1_tools_1_1_input_parser}{Input\+Parser}} class provides an interface to input parameter files. The class can be used as follows. ~\newline
 Suppose you have a file input\+\_\+file.\+prm containing the following text

\begin{DoxyVerb}  # Define the mesh parameters

  xmax = 10.0 # Size of the mesh along the x-axis, in micrometers.
  ymax = 10.0 # Size of the mesh along the y-axis, in micrometers.
  zmax = 10.0 # Size of the mesh along the z-axis, in micrometers.

  nx = 100 # number of points along the x-axis
  ny = 100 # number of points along the y-axis
  nz = 100 # number of points along the z-axis

  # Define interaction parameters

  scattering length = 100.0 # Scattering length in micrometers

  # Define the run mode

  calculate ground state = true # If true, calculate a ground state solution of the eGPE\end{DoxyVerb}


Comments are defined by an hashtag (\#) and ignored by the class. ~\newline
Every line containing an equal (=) sign is interpreted as a line defining an input. Every character on the left of the equal sign is interpreted as the name of the variable (without blanks), while what follows the equal sign on the right is interpreted as the value of the defined variable. ~\newline
In order to retrieve the value of these variables and use them in the code, it is necessary to use the appropriate member function {\ttfamily $<$variable type$>$ Input\+Parser\+::retrieve\+\_\+$<$type$>$(\char`\"{}variable name\char`\"{}) }, which will cast the retrieved variable (which is first interpreted as an {\ttfamily std\+::string}) to the desired {\ttfamily  $<$type$>$ }. ~\newline
